# 手工调试文档

当前手工调试默认以 Windows 工作区和真实硬件为准。

当前已经验证通过的一套原型环境是：

- 摄像头：手机摄像头
- 串口：`CH340`
- 舵机供电：`CH340` 临时供电

这套配置可以完成课程作业所需的核心功能闭环，但不是推荐的最终硬件形态。当前最终硬件方案已经加入红绿 LED 和蜂鸣器；若继续收尾，优先验证状态提示硬件、独立固定摄像头和独立 5V 舵机供电。

## 1. 构建与烧录

```bash
uv sync
uv run pio run --project-dir firmware
uv run pio run --project-dir firmware -t upload
```

## 2. 查看串口

```bash
uv run python -m serial.tools.list_ports -v
```

确认 `COMx` 后，再做串口联调。

## 3. 串口最小联调

```bash
uv run python scripts/pointer_serial_cli.py --port COM4 ping
uv run python scripts/pointer_serial_cli.py --port COM4 status
uv run python scripts/pointer_serial_cli.py --port COM4 center
uv run python scripts/pointer_serial_cli.py --port COM4 stop
```

检查点：

- `PING` 返回 `PONG`
- `STATUS?` 返回 `STATUS:...`
- `CENTER` 和 `STOP` 返回 `OK:*`
- `STATUS?` 中包含 `MODE=IDLE`、`MODE=SEARCH`、`MODE=LOCK` 或 `MODE=LOST`

## 4. 状态硬件联调

```bash
uv run python scripts/pointer_serial_cli.py --port COM4 state search
uv run python scripts/pointer_serial_cli.py --port COM4 state lock
uv run python scripts/pointer_serial_cli.py --port COM4 state lost
uv run python scripts/pointer_serial_cli.py --port COM4 state idle
```

检查点：

- `SEARCH`：绿 LED 慢闪，蜂鸣器静音
- `LOCK`：绿 LED 常亮，蜂鸣器响 1 声
- `LOST`：红 LED 常亮，蜂鸣器响 2 声
- `IDLE`：红绿 LED 熄灭，蜂鸣器静音

## 5. 摄像头检查

```bash
uv run python scripts/pointer_vision_app.py --list-cameras
```

如果 Windows 默认打开失败，优先尝试 `msmf`，再看是否需要 `dshow`。

## 6. 桌面端演示入口

```bash
uv run python scripts/pointer_desktop_app.py --port COM4 --camera 0 --camera-backend msmf --model yolov8n.pt
```

建议按这条顺序检查：

1. 连接串口
2. 打开摄像头
3. 点击一个检测框或拖框初始化
4. 观察箭头是否持续跟随，同时确认绿 LED 常亮
5. 遮挡或离开目标，确认是否按配置执行 `STOP` 或 `CENTER`，并确认红 LED 常亮
6. 断开或退出时确认设备进入 `IDLE`

## 7. 命令行视觉调试入口

```bash
uv run python scripts/pointer_vision_app.py --port COM4 --camera 0 --camera-backend msmf --model yolov8n.pt --verbose
```

这个入口主要用于排查摄像头、识别和串口联动，不作为默认演示入口。
