# 项目概述

## 产品目标

TargetPointer 是一套固定摄像头下的实体人物指向演示装置。操作者在实时画面中选中一个人，上位机持续维护该目标，并通过串口驱动 Blue Pill + SG90，让实体箭头在水平方向指向目标；红绿 LED 和蜂鸣器用于提示搜索、锁定、丢失和空闲状态。

成功标准很直接：第一次看到的人能理解“选中谁，箭头就指谁”；演示者能用少量步骤完成“连接设备 -> 打开相机 -> 选中人物 -> 持续指向 -> 丢失后停止或回中”的闭环；设备动作可预测，不出现随机角度输出。

## 当前范围

- Windows 原生环境是实机运行权威环境，WSL 只作为代码镜像。
- 固定摄像头输入，当前可用手机摄像头或 USB 摄像头，演示时不频繁移动机位。
- YOLO `person` 检测用于初始化候选，用户可点击检测框或拖框手动初始化。
- 单目标人物维护，通过几何重关联和短时重获保持目标。
- 目标水平偏移映射为 SG90 单轴舵机角度，并执行死区、分段限速和最小发送阈值。
- Blue Pill 固件解析串口文本命令，约束角度边界，驱动 SG90、红绿 LED 和蜂鸣器。
- 桌面端提供主控台、活动日志、数据分析、中文报告和 LiveKit 实时语音对话入口。

## 当前不做

- 多目标身份管理、ReID、上传参考图找人。
- 双轴云台、三维定位、距离估计。
- 在 Blue Pill 端做视觉识别或目标语义判断。
- 与当前演示主线无关的额外输入模式或外围硬件。

## 软件结构

- `scripts/pointer_desktop_app.py`：默认演示入口，负责 UI 操作流、摄像头选择、串口连接、目标选择、报告、语音和状态展示。
- `targetpointer/runtime/runtime.py`：摄像头读取、检测结果处理、目标维护、串口交互和状态快照。
- `targetpointer/vision/app.py`：命令行视觉调试入口，也提供角度计算等运行时共用逻辑。
- `targetpointer/runtime/host_logic.py`：纯控制逻辑，包括偏移映射、死区、限速、发送阈值和丢失策略。
- `targetpointer/runtime/serial.py`：串口命令发送、响应读取和状态查询封装。
- `targetpointer/reporting/report.py`：目标裁剪、全景图、OpenAI 视觉分析和中文 PDF 报告生成。
- `targetpointer/voice/agent.py`：LiveKit worker 入口，使用 `VAD + STT + LLM + TTS` pipeline 进行实时语音对话。
- `firmware/src/main.cpp`：Blue Pill 固件，负责命令解析、舵机、LED、蜂鸣器和状态响应。

## 实时语音

语音模块按 LiveKit 的 worker 模式运行。桌面端负责启动 `scripts/pointer_voice_agent.py dev`，worker 连接用户提供的 LiveKit server/room，作为实时参与者处理音频和文本流。正常使用桌面端时不需要手动单独启动 worker；如果要独立调试，可在 Windows 侧运行：

```powershell
uv run python scripts/pointer_voice_agent.py dev
```

实时会话默认使用 pipeline 模式：Silero VAD、OpenAI `gpt-4o-mini-transcribe` 中文转写、OpenAI `gpt-4o-mini` LLM、ElevenLabs `eleven_turbo_v2_5` TTS 和可配置 voice id。这样可以独立调整 VAD、STT、LLM、TTS 和音色。噪声环境下 VAD 阈值默认提高到 `0.75`，静音判定默认 `450 ms`。桌面端提供“静音用户麦克风”开关，它会让 worker 暂停接收用户音频，同时保留 AI 输出能力。字幕由 worker 写入 `runtime/voice/transcript.jsonl`，桌面端以固定滚动区域展示。

## 报告

报告模块只描述可见信息，不做身份识别。提示词要求输出中文，并尽量覆盖背景环境、人物穿着、姿态、相对位置、可见物品、当前活动、画面质量、系统状态和不确定性。PDF 使用中文字体，避免中文内容乱码。

## 角度方向

当前摄像头画面是镜像画面，实际舵机旋转方向与原映射相反。上位机在输出目标角前已围绕中心角做反向处理：画面左侧目标会映射到更大的舵机角，画面右侧目标会映射到更小的舵机角。

## 推荐演示流程

1. 接好 `Blue Pill + CH340 + SG90 + 红绿 LED + 蜂鸣器`，先用串口命令验证 `PING`、`STATUS?`、`CENTER`、`STOP`、`STATE:SEARCH`。
2. 在 Windows 工作区启动桌面端：

```powershell
uv run python scripts/pointer_desktop_app.py --camera 0 --camera-backend msmf --model yolov8n.pt
```

3. 在主界面确认串口下拉框显示 `COM4` 或实际端口；只有显式传 `--port COMx` 或 `--auto-connect` 时才会启动即连接。
4. 扫描并打开摄像头，在实时画面里点击检测到的人物，必要时拖框手动初始化。
5. 观察舵机、LED、蜂鸣器、Activity、Insights、报告和语音对话是否符合预期。

## 当前风险与优先级

- 长时间运行时，SG90 应优先使用独立稳定 5V 供电，避免依赖 CH340 临时供电。
- 最终展示前需要固定摄像头机位、舵机支架和箭头结构，减少抖动。
- 噪声环境下仍需在 Windows 实机上复测实时语音 VAD、字幕流和静音控制。
- 镜像摄像头下的舵机方向已在代码侧反转，仍需要实机重新标定最小角、中心角和最大角。
