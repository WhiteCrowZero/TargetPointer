# 项目概述

## 项目定位

TargetPointer 是一个“固定摄像头下的实体人物指向系统”。

系统在固定摄像头视角下，使用 YOLO 辅助目标人物的检测与初始化，由上位机持续维护目标位置，并通过串口驱动下位机舵机，实现对目标人物的实体箭头水平指向。

## 产品故事

当前版本更适合被定义为一套**互动演示装置**，而不是复杂的视觉识别产品。

它要传达的体验是：

- 用户在画面中选中一个人
- 系统理解这个选择
- 实体箭头在现实中持续指向这个人

因此，当前产品价值首先是“直观”和“可演示”，其次才是底层算法细节。

## 当前目标

- 用固定摄像头观察单一场景。
- 只处理单目标人物指向。
- 由上位机完成视觉、控制和串口联动。
- 由 Blue Pill 执行 SG90 的单轴水平动作。
- 以“尽快可演示”为第一优先级。

## 当前完成情况

- 已完成单主线收缩，旧语音和旧类别物体叙事已移除。
- 已完成 YOLO 人物检测初始化、单目标持续定位和串口控制主链路。
- 已完成 Windows 优先的桌面端演示界面，支持摄像头选择、串口连接、点击选人和手动框选。
- 已完成基础丢失策略和自适应分段限速舵机控制，避免初始大偏差时角度突跳。
- 当前主要剩余工作是独立 5V 条件下的实机联调和角度参数标定，而不是再扩展功能边界。
- 当前产品路线已经明确：先把它做成稳定的互动演示装置，再决定是否继续走结构样机化。

## 使用场景

- 室内固定摄像头机位。
- 演示对象为画面中的人物，而不是桌面物体类别。
- 目标人物的初始化由用户在实时画面中完成。
- 当前只要求水平指向，不追求空间定位或复杂身份识别。

## 核心思路

- YOLO 负责人物检测与初始化支持。
- 用户优先通过点击一个 YOLO 检测到的人物来选定目标。
- 当检测框不理想时，也允许手动框选人物区域作为初始化输入。
- 初始化完成后，由上位机的后续跟踪/位置更新逻辑持续维护目标框。
- 上位机根据目标框中心点与画面中心的水平偏移计算目标角度，再通过自适应分段限速生成实际输出角度。
- Blue Pill 只接收串口命令并控制 SG90，不参与任何识别决策。

## 唯一工作流程

1. 启动固定摄像头并显示实时画面。
2. YOLO 在画面中检测人物，为初始化提供候选框。
3. 用户点击一个检测到的人物，或手动框选人物区域。
4. 上位机以该目标初始化跟踪/持续定位。
5. 上位机持续计算目标在画面中的水平位置。
6. 上位机把水平偏移映射为舵机角度，通过串口发送给 Blue Pill。
7. Blue Pill 驱动 SG90，带动实体箭头持续指向目标人物。
8. 目标连续丢失若干帧后，系统执行 `STOP` 或 `CENTER`。

## 当前边界

- 不是物体类别指向系统。
- 不是上传参考图找某个人的系统。
- 不是多目标身份识别或 ReID 系统。
- 不是双轴云台或空间定位系统。
