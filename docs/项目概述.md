# 项目概述

## 产品定位

TargetPointer 是一套固定摄像头下的实体人物指向演示装置。

它要表达的核心体验是：

- 用户在屏幕里选中一个人
- 系统理解这个选择并持续维护目标
- 现实中的实体箭头持续指向这个人

当前版本优先追求直观、可重复、步骤少，而不是扩展复杂算法边界。

## 当前范围

- 固定摄像头输入
- YOLO `person` 检测作为初始化入口
- 单目标人物维护
- 水平偏移到舵机角度映射
- Blue Pill + SG90 单轴水平指向
- Windows 优先的桌面端演示入口

## 当前不做

- 多目标身份管理
- ReID 或上传参考图找人
- 双轴云台
- 三维定位或距离估计
- 与当前演示主线无关的额外输入模式

## 推荐演示流程

1. 接好 `Blue Pill + CH340 + SG90`，串口先验证 `PING`、`STATUS?`、`CENTER`、`STOP`。
2. 启动 `scripts/pointer_desktop_app.py`。
3. 连接串口，打开当前用于演示的摄像头。
4. 在实时画面中点击一个检测到的人物，必要时拖框手动初始化。
5. 系统持续维护该目标，并把水平偏移转换为舵机角度。
6. 目标丢失后按配置执行 `STOP` 或 `CENTER`。

## 当前软件状态

- 桌面端是默认演示入口，提供视频舞台、串口/摄像头连接、点击选人、拖框初始化、活动日志和分析页。
- 主界面会给出下一步推荐动作，并按当前状态禁用不该点的按钮。
- 失败操作会显示短暂错误提示，详细信息写入 `Activity`。
- 视觉与控制链路已经实现基础重关联、丢失策略和分段限速输出角控制。
- 固件已支持状态查询、渐进回中、位置保持、板载 LED 阶段提示和配置化步进参数。

## 当前验证结论

- Windows 侧已经完成真实链路验证，当前可以实际完成“打开视频流 -> 选中人物 -> 箭头持续指向 -> 丢失后停止或回中”的闭环。
- 当前验证使用的是原型配置：视频输入来自手机摄像头，舵机由 `CH340` 提供短时供电。
- 这说明课程作业所需的核心功能已经完成；独立固定摄像头和独立 5V 舵机供电属于后续工程化优化项。

## 当前主要风险

- 长时间运行下，`CH340` 临时供电的稳定性和安全余量仍弱于独立 5V 方案。
- 当前手机摄像头可以完成演示，但还不是最终想定的独立固定摄像头形态。
- 若作为最终展示样机提交，仍建议补独立供电、固定机位和一轮参数复标定。
