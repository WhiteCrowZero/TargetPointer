# 架构设计

## 总体架构

“寻音指针”采用“上位机负责感知与决策，Blue Pill 负责执行”的分层架构。系统按功能分为五层：

## 视频输入层

- 输入源：手机通过 USB 接入电脑，作为摄像头提供实时视频流。
- 输出：连续视频帧，供上位机检测模块使用。

## 上位机感知层

- 核心能力：YOLO 目标检测。
- 输入：视频帧和当前待查找目标类别。
- 输出：目标框、类别、置信度和目标框中心点坐标。

## 上位机交互层

- 核心能力：语音识别与目标名称解析。
- 输入：用户语音。
- 输出：目标类别名称，例如 `cup`、`keys`。
- 职责：将自然语言输入收敛为可供检测模块使用的目标类别。

## 控制层

- 核心能力：角度计算与串口协议发送。
- 输入：目标框中心点、画面宽度、当前映射参数。
- 输出：串口文本命令，例如 `ANGLE:108`。
- 职责：把画面中的目标水平位置转换为舵机目标角度，并发送给 Blue Pill。

## 执行层

- 硬件：Blue Pill + SG90。
- 输入：串口命令。
- 输出：舵机转角动作。
- 职责：解析文本命令、校验角度范围、控制 SG90 转到目标角度或回中。

## 推荐数据流

1. 手机通过 USB 输出视频流到电脑。
2. 用户在电脑端说出目标名称。
3. 语音识别模块输出目标类别。
4. YOLO 在视频流中检测该类别并定位目标框。
5. 上位机计算目标框中心点相对画面中心的水平偏移。
6. 控制层将偏移换算为目标方向，再映射为舵机角度。
7. 上位机通过串口向 Blue Pill 发送 `ANGLE:<deg>`。
8. Blue Pill 控制 SG90 带动箭头转向目标。

## 角度计算思路

设画面宽度为 `W`，目标框中心横坐标为 `x`，画面中心为 `W / 2`。

- 先计算水平偏移量：`offset = x - W / 2`
- 再计算归一化偏移：`ratio = offset / (W / 2)`
- 根据摄像头水平视场角，将 `ratio` 换算为目标方向偏角
- 最后把目标方向偏角映射到舵机安全角度范围 `20° ~ 160°`

V1 可以先用线性映射，后续再按实际安装位置做标定修正。

## 为什么选择上位机方案

相比旧的独立外设识别与定位思路，上位机方案更适合当前仓库目标：

- 视觉识别放在电脑端，更容易运行 YOLO，不受 Blue Pill 算力限制。
- 语音识别放在电脑端，实现更直接，便于后续更换方案。
- 手机摄像头接入简单，适合桌面原型验证。
- Blue Pill 只保留串口与舵机控制，固件边界清晰，调试成本更低。
- 无需额外制作标签或接收器，系统结构更接近真实“看见并指向”的演示效果。

## 实现边界

- Blue Pill 不负责目标检测、语音识别和图像处理。
- 上位机不直接控制 PWM，而是只发送高层文本命令。
- 当前系统默认单摄像头、单舵机、单轴水平指向。
