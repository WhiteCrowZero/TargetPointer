# 架构设计

## 总体结构

当前系统采用“Windows 上位机负责感知与控制，Blue Pill 负责执行”的最小闭环架构。

## 入口层

- `scripts/pointer_desktop_app.py`
  当前默认演示入口。负责操作流、主界面展示、活动日志和 Insights 分析页。
- `scripts/pointer_vision_app.py`
  调试入口。用于摄像头、识别和串口链路的命令行式验证。
- `scripts/pointer_serial_cli.py`
  协议与串口联调工具。

## 上位机运行时

- `scripts/pointer_runtime.py`
  负责摄像头读取、检测结果处理、目标维护、串口交互和状态快照。
- `scripts/pointer_host_logic.py`
  负责纯控制逻辑，包括偏移映射、死区、分段限速、最小发送阈值和丢失策略。
- `scripts/pointer_serial.py`
  负责命令发送、响应读取和状态查询封装。

## 视觉与目标维护

- 检测层使用 YOLO 的 `person` 结果作为初始化候选。
- 选中目标后，运行时通过几何重关联和短时重获继续维护单目标。
- 当前不引入复杂身份特征或多目标调度。

## 控制链路

1. 摄像头产生实时帧。
2. 视觉层给出人物候选框。
3. 用户点击检测框或拖框完成初始化。
4. 运行时持续更新目标框。
5. 控制层把目标水平偏移换算成目标角，再生成实际输出角。
6. 串口层发送 `ANGLE`、`CENTER`、`STOP`、`LED` 和 `STATUS?`。
7. 固件按安全边界执行 SG90 动作并返回状态。

## 桌面端职责

- 提供默认演示操作流。
- 持续显示下一步推荐动作。
- 按状态禁用不合时宜的按钮。
- 在失败时显示短暂错误提示，并把详细记录留在 `Activity`。

桌面端不重写视觉算法和协议，只消费运行时状态并组织交互。

## 固件职责

- 解析串口文本命令。
- 维护当前角度、目标角、LED 状态和最近一次结果。
- 按配置步进控制舵机，支持渐进回中和位置保持。
- 通过 `STATUS` 响应把当前设备状态回传上位机。

固件不负责人物识别，也不保留目标语义。
