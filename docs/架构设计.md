# 架构设计

## 总体架构

当前版本采用“上位机负责感知与控制，Blue Pill 负责执行”的最小闭环架构，围绕单目标人物指向展开。

## 视频输入层

- 输入源：USB 摄像头或手机摄像头。
- 约束：机位固定，视角稳定。
- 输出：连续视频帧。

## 上位机视觉层

- 核心能力：YOLO 人物检测。
- 作用：为目标初始化提供 `person` 候选框。
- 推荐流程：用户在实时画面中点击一个 YOLO 检测到的人物。
- 兜底流程：用户手动框选人物区域。

## 上位机目标维护层

- 核心能力：单目标持续定位。
- 输入：初始化得到的目标框。
- 输出：每一帧或每一轮更新后的目标框中心点。
- 当前实现为“YOLO 持续检测 + 单目标几何重关联 + 短时重获”，不引入复杂 ReID。

## 上位机控制层

- 核心能力：把目标水平位置转换为舵机控制量。
- 输入：目标框中心横坐标、画面宽度、控制参数。
- 输出：`ANGLE:<deg>`、`STOP` 或 `CENTER`。
- 纯逻辑职责：偏移映射、死区、自适应分段限速、最小发送阈值、丢失策略。

## 上位机演示层

- 形态：Windows 优先的单窗口桌面应用。
- 能力：摄像头选择、串口连接、点击选人、手动框选、状态展示、活动日志查看。
- 边界：桌面端只负责操作体验和状态呈现，不改变视觉识别与串口协议逻辑。

## 固件执行层

- 硬件：Blue Pill + SG90。
- 输入：串口文本命令。
- 输出：实体箭头的水平转向动作和状态回报。
- 边界：固件不做人物识别，也不保存目标语义，只执行方向控制。

## 推荐数据流

1. 摄像头向上位机提供实时视频帧。
2. YOLO 在画面中检测人物，输出候选框。
3. 用户选择一个人物候选框，完成初始化。
4. 上位机持续更新该目标的人物框位置。
5. 控制层根据目标框中心点与画面中心的水平偏移计算目标角度。
6. 控制层再把目标角转换为带安全限速的实际输出角度。
7. 上位机将控制结果通过串口发送给 Blue Pill。
8. Blue Pill 控制 SG90 驱动实体箭头持续指向目标人物。
9. 若目标连续丢失若干帧，则发出 `STOP` 或 `CENTER`。

## 角度计算思路

设画面宽度为 `W`，目标框中心横坐标为 `x`，画面中心为 `W / 2`。

- 水平偏移：`offset = x - W / 2`
- 归一化偏移：`ratio = offset / (W / 2)`
- 根据舵机可用角度范围，将 `ratio` 线性映射到目标角范围 `20° ~ 160°`
- 目标角先经过中心死区处理，避免中心附近频繁修正
- 再根据当前输出角与目标角的误差，进入“小步进 / 中步进 / 大步进”三段限速控制
- 最终输出实际发送给舵机的安全角度，避免初始大偏差时突然猛跳

当前版本优先使用“线性目标角 + 分段限速输出角”，后续再依据实机安装位置做标定微调。

## 为什么保留 YOLO

- 人物检测与初始化阶段，YOLO 比纯手动框选更快、更适合演示。
- 固定摄像头下，先用 YOLO 找人，再进入单目标持续更新，复杂度可控。
- 当前目标是“尽快稳定演示”，不需要引入复杂多目标身份系统。

## 实现边界

- 上位机负责全部目标识别逻辑。
- 固件只处理方向控制和设备状态。
- 当前系统默认单摄像头、单目标、单舵机、单轴水平指向。
