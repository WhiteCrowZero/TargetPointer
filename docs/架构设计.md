# 架构设计

## 总体架构

当前版本采用“上位机负责感知与控制，Blue Pill 负责执行”的最小闭环架构，围绕单目标人物指向展开。

## 视频输入层

- 输入源：USB 摄像头或手机摄像头。
- 约束：机位固定，视角稳定。
- 输出：连续视频帧。

## 上位机视觉层

- 核心能力：YOLO 人物检测。
- 作用：为目标初始化提供 `person` 候选框。
- 推荐流程：用户在实时画面中点击一个 YOLO 检测到的人物。
- 兜底流程：用户手动框选人物区域。

## 上位机目标维护层

- 核心能力：单目标持续定位。
- 输入：初始化得到的目标框。
- 输出：每一帧或每一轮更新后的目标框中心点。
- 当前实现可以采用“检测初始化 + 跟踪/位置更新”的轻量组合，不引入复杂 ReID。

## 上位机控制层

- 核心能力：把目标水平位置转换为舵机控制量。
- 输入：目标框中心横坐标、画面宽度、控制参数。
- 输出：`ANGLE:<deg>`、`STOP` 或 `CENTER`。
- 纯逻辑职责：偏移映射、死区、平滑、丢失策略。

## 固件执行层

- 硬件：Blue Pill + SG90。
- 输入：串口文本命令。
- 输出：实体箭头的水平转向动作和状态回报。
- 边界：固件不做人物识别，也不保存目标语义，只执行方向控制。

## 推荐数据流

1. 摄像头向上位机提供实时视频帧。
2. YOLO 在画面中检测人物，输出候选框。
3. 用户选择一个人物候选框，完成初始化。
4. 上位机持续更新该目标的人物框位置。
5. 控制层根据目标框中心点与画面中心的水平偏移计算目标角度。
6. 上位机将控制结果通过串口发送给 Blue Pill。
7. Blue Pill 控制 SG90 驱动实体箭头持续指向目标人物。
8. 若目标连续丢失若干帧，则发出 `STOP` 或 `CENTER`。

## 角度计算思路

设画面宽度为 `W`，目标框中心横坐标为 `x`，画面中心为 `W / 2`。

- 水平偏移：`offset = x - W / 2`
- 归一化偏移：`ratio = offset / (W / 2)`
- 根据舵机可用角度范围，将 `ratio` 线性映射到 `20° ~ 160°`
- 再叠加死区与平滑策略，减少抖动

当前版本优先使用线性映射，后续再依据实机安装位置做标定微调。

## 为什么保留 YOLO

- 人物检测与初始化阶段，YOLO 比纯手动框选更快、更适合演示。
- 固定摄像头下，先用 YOLO 找人，再进入单目标持续更新，复杂度可控。
- 当前目标是“尽快稳定演示”，不需要引入复杂多目标身份系统。

## 实现边界

- 上位机负责全部目标识别逻辑。
- 固件只处理方向控制和设备状态。
- 当前系统默认单摄像头、单目标、单舵机、单轴水平指向。
