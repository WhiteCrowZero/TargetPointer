# 硬件调试记录

## 当前状态说明

本项目当前统一为“固定摄像头 + 上位机 YOLO 检测与控制 + Blue Pill + SG90 + 红绿 LED + 蜂鸣器”的单主线方案。旧的物体类别指向叙事和其他外围方案不再作为调试依据。

## 当前实测结论

- Windows 侧真实链路已跑通。
- 当前可完成：打开摄像头、连接串口、选择人物、持续指向、丢失后 `STOP` / `CENTER`。
- 当前验证使用的是原型环境，而不是最终硬件形态：
  - 摄像头：手机摄像头
  - 舵机供电：`CH340` 临时供电
- 因此，当前应把项目状态定义为“核心功能已完成并通过原型实测，硬件工程化仍可继续优化”。

## 当前硬件盘点

### 已有硬件

- `Blue Pill`
- `STM32F103 核心板`
- `SG90`
- `ST-LINK V2`
- `CH340 USB-TTL`
- `HW-131`
- `面包板`
- `杜邦线`
- `电阻`
- `红绿 LED`
- `蜂鸣器模块`

### 说明

- `Blue Pill` 是当前正式主控。
- `SG90` 是当前唯一执行机构。
- 红绿 LED 用于远距离状态提示；蜂鸣器用于锁定和丢失提示。
- 当前结构默认服务于固定摄像头下的单轴水平指向。

## 待验证项

- 独立 5V 供电下的长时间稳定性
- 独立固定摄像头替换后的机位稳定性
- 当前原型环境下的长时间连续运行稳定性
- 镜像摄像头下，反向后的水平偏移到舵机角度最终标定参数
- `STATE:IDLE/SEARCH/LOCK/LOST` 下红绿 LED 和蜂鸣器状态测试
- LiveKit 实时语音在嘈杂环境下的 VAD 阈值、字幕滚动和用户静音验证

## 最近一次有效验证环境

- 主控：`Blue Pill`
- 串口：`CH340 USB-TTL`
- 执行机构：`SG90`
- 摄像头：手机摄像头
- 舵机供电：`CH340` 临时供电
- 结果：核心闭环功能可正常运行

## 2026-04-23 镜像侧变更记录

- 环境：WSL 代码镜像，未连接实际串口、摄像头和 LiveKit server。
- 变更：上位机角度输出按镜像摄像头做反向处理；桌面端串口默认显示 `COM4`，但不再在未显式传 `--port` 或 `--auto-connect` 时自动连接。
- 变更：语音 worker 改为 `VAD + STT + LLM + TTS` pipeline，默认中文、`gpt-4o-mini-transcribe`、`gpt-4o-mini`、ElevenLabs `eleven_turbo_v2_5`、可配置 voice id、Silero VAD 阈值 `0.75`、静音判定 `450 ms`。
- 变更：桌面端增加用户麦克风静音控制、实时字幕滚动区和 worker 状态动画。
- 待实机确认：Windows 侧 LiveKit server 接入、噪声环境 VAD、字幕流、静音控制、镜像角度方向和舵机最终标定。
