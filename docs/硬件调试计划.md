# 硬件调试计划

## 1. Blue Pill + ST-LINK 下载验证

- 目的：确认主控板可正常烧录和运行最小程序。
- 输入：Blue Pill、ST-LINK V2、最小点灯或串口打印程序。
- 预期输出：程序可成功下载，板卡可复位后稳定运行。
- 风险点：SWD 接线错误、供电不稳、板卡本体异常。

## 2. SG90 舵机转角测试

- 目的：确认 PWM 控制有效，并验证安全转角范围。
- 输入：Blue Pill、SG90、5V 供电、固定角度测试程序。
- 预期输出：舵机可在 `20° ~ 160°` 范围内稳定转动，并能回到 `90°`。
- 风险点：供电不足、抖动、卡死、机械安装过紧。

## 3. CH340 串口通信测试

- 目的：确认上位机与 Blue Pill 串口链路正常。
- 输入：CH340 USB-TTL、Blue Pill 串口回显或日志程序。
- 预期输出：电脑端可稳定发送和接收 `115200 8N1` 文本数据。
- 风险点：TX/RX 接反、串口号变化、地线未共地、电平连接错误。

## 4. 上位机串口发 ANGLE 命令测试

- 目的：确认文本协议和舵机控制链路已打通。
- 输入：上位机串口工具或脚本、`ANGLE:<deg>` / `CENTER` / `STOP` 命令。
- 预期输出：Blue Pill 能正确解析命令并驱动 SG90 响应。
- 风险点：命令格式不一致、换行处理错误、角度边界未校验。

## 5. 手机摄像头接入电脑测试

- 目的：确认手机可通过 USB 作为摄像头稳定输入视频。
- 输入：手机、USB 数据线、电脑端摄像头接入方案。
- 预期输出：电脑端可持续获取实时视频画面。
- 风险点：手机端模式不兼容、驱动问题、画面延迟过大。

## 6. YOLO 视频检测测试

- 目的：确认上位机能在视频流中检测目标类别。
- 输入：视频流、YOLO 模型、待检测目标样本。
- 预期输出：可输出目标框、类别和置信度。
- 风险点：模型体积过大、帧率不足、目标类别效果不稳定。

## 7. 目标中心点到角度映射测试

- 目的：验证画面坐标到舵机角度的换算逻辑。
- 输入：目标框中心点坐标、画面宽度、舵机角度边界。
- 预期输出：目标越靠左，舵机角度越偏左；目标越靠右，舵机角度越偏右；中心目标对应接近 `90°`。
- 风险点：摄像头视场角估计不准、安装角度偏差、线性映射误差。

## 8. 整机联调

- 目的：完成“语音输入 -> 视觉检测 -> 角度计算 -> 串口发令 -> 舵机转向”的完整闭环。
- 输入：手机视频流、上位机程序、Blue Pill、SG90、固定桌面目标。
- 预期输出：用户指定目标后，箭头能在桌面场景中完成一次稳定指向。
- 风险点：多模块时序不一致、串口延迟、识别抖动、供电扰动导致复位。
