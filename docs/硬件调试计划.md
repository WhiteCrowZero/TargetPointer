# 硬件调试计划

## 1. Blue Pill + ST-LINK 下载验证

- 目的：确认主控板可正常烧录和运行最小程序。
- 输入：Blue Pill、ST-LINK V2、当前固件。
- 预期输出：程序可成功下载，板卡复位后稳定启动。
- 风险点：SWD 接线错误、供电不稳、板卡异常。

## 2. SG90 舵机转角测试

- 目的：确认 PWM 控制有效，并验证安全转角范围。
- 输入：Blue Pill、SG90、独立 5V 供电。
- 预期输出：舵机可在 `20° ~ 160°` 范围内稳定转动，并能回到 `90°`。
- 风险点：供电不足、结构卡滞、持续抖动。

## 3. CH340 串口通信测试

- 目的：确认上位机与 Blue Pill 串口链路正常。
- 输入：CH340 USB-TTL、Blue Pill。
- 预期输出：电脑端可稳定发送和接收 `115200 8N1` 文本数据。
- 风险点：TX/RX 接反、未共地、串口号变化。

## 4. 上位机串口发令测试

- 目的：确认文本协议和舵机控制链路已打通。
- 输入：`PING`、`STATUS?`、`CENTER`、`STOP`、`ANGLE:<deg>`。
- 预期输出：Blue Pill 能正确解析命令并驱动 SG90 响应。
- 风险点：命令格式错误、换行处理错误、角度边界未校验。

## 5. 固定摄像头接入测试

- 目的：确认 USB 摄像头或手机摄像头可稳定接入上位机。
- 输入：摄像头、数据线、上位机。
- 预期输出：程序可持续获取实时视频画面。
- 风险点：驱动问题、画面延迟过大、视频源不稳定。

## 6. YOLO 人物检测测试

- 目的：确认上位机能在视频流中检测人物。
- 输入：实时视频流、YOLO 模型。
- 预期输出：画面中可输出 `person` 检测框。
- 风险点：帧率不足、漏检、误检。

## 7. 目标初始化与持续更新测试

- 目的：确认用户可选择一个人物，并建立后续位置更新链路。
- 输入：YOLO 检测结果或手动框选。
- 预期输出：目标框被初始化后，系统能持续维护单目标位置。
- 风险点：选中错误目标、目标丢失过快、初始化框不稳定。

## 8. 水平偏移到角度映射测试

- 目的：验证画面坐标到舵机角度的换算逻辑。
- 输入：目标框中心点坐标、画面宽度、舵机角度边界。
- 预期输出：左侧目标对应较小角度，中间接近 `90°`，右侧对应较大角度。
- 风险点：安装偏差、线性映射误差、中心定义不准。

## 9. 整机联调

- 目的：完成“固定摄像头 -> 选人 -> 持续指向 -> 丢失处理”的完整闭环。
- 输入：实时视频流、上位机程序、Blue Pill、SG90、固定场景人物。
- 预期输出：实体箭头能持续水平指向目标人物，丢失后执行 `STOP` 或 `CENTER`。
- 风险点：识别抖动、串口时序问题、舵机供电扰动导致复位。
