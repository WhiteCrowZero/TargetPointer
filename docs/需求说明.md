# 需求说明

## 项目目标

在固定摄像头场景下，用户从上位机实时画面中选择一个人物目标，系统借助 YOLO 完成检测与初始化，并通过串口驱动 Blue Pill + SG90，让实体箭头持续做单轴水平指向。

## 产品目标

当前版本的产品目标不是做成复杂视觉平台，而是做成一套**低复杂度、可重复演示的互动装置**。

产品层的成功标准是：

- 第一次看到的人能快速理解“选中谁，箭头就指谁”
- 演示者能用很少步骤完成一次闭环展示
- 实体动作稳、克制、可预测

## 功能需求

1. 上位机能够读取固定摄像头实时画面。
2. YOLO 能够在画面中检测 `person`，为目标初始化提供候选框。
3. 用户能够点击一个检测到的人物，或手动框选人物区域。
4. 上位机能够在初始化后持续维护单目标位置。
5. 上位机能够根据目标框中心点与画面中心的水平偏移计算控制角度。
6. 上位机能够对角度输出做基础死区、平滑和丢失策略处理。
7. 上位机能够通过串口向 Blue Pill 发送 `ANGLE:<deg>`、`CENTER`、`STOP`、`STATUS?`。
8. Blue Pill 能够接收串口命令并驱动 SG90 在安全角度范围内动作。
9. 目标丢失若干帧后，系统能够执行 `STOP` 或 `CENTER`。

## 非功能需求

- 摄像头机位固定，不在运行中频繁移动。
- 当前只支持单目标人物，不做多目标身份管理。
- Blue Pill 端保持简单，只负责串口协议和舵机动作执行。
- 文档、依赖和运行说明只围绕视觉、串口和固件主线。
- 安装与运行过程只围绕视觉、串口和固件环境。
- 演示链路要优先追求可复现、低复杂度和易调试。
- 主界面和主流程优先服务于演示体验，而不是工程调试完整性。

## 当前版本范围内

- USB 摄像头或手机摄像头接入上位机
- YOLO 人物检测
- 单目标人物初始化
- 初始化后的持续位置更新
- 水平偏移到舵机角度映射
- 串口文本协议控制
- SG90 单轴水平指向

## 当前版本范围外

- 额外输入模态
- 参考图找人
- ReID 和身份识别
- 多目标目标管理
- 双轴云台
- 三维定位和距离估计
- 完整产品化硬件定制

## 验收重点

- 能稳定显示实时画面并检测人物。
- 用户能选中一个人物并完成初始化。
- 箭头能随目标人物的水平移动持续调整方向。
- 丢失目标时系统行为可预测，执行 `STOP` 或 `CENTER`。
- 固件和上位机职责边界清晰，演示链路可重复跑通。
