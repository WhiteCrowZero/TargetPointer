# 需求说明

## 目标

在固定摄像头场景下，用户从实时画面中选择一个人物目标，系统持续维护该目标，并通过串口驱动 Blue Pill + SG90，让实体箭头做单轴水平指向。

## 产品成功标准

- 第一次看到的人能快速理解“选中谁，箭头就指谁”。
- 演示者可以用较少步骤完成一次闭环演示。
- 设备动作可预测，丢失目标后的行为明确。

## 功能需求

1. 上位机能够读取固定摄像头实时画面。
2. 系统能够检测 `person` 并提供可选目标框。
3. 用户能够点击检测框选人，也能够拖框手动初始化。
4. 初始化后系统能够持续维护单目标位置。
5. 上位机能够把目标水平偏移映射到安全角度范围内的舵机目标角。
6. 上位机能够执行死区、分段限速、最小发送阈值和丢失策略。
7. 上位机能够通过串口发送 `ANGLE:<deg>`、`CENTER`、`STOP`、`STATUS?`、`LED:ON`、`LED:OFF`。
8. 固件能够解析命令、约束角度边界、渐进回中并返回设备状态。

## 交互需求

1. 默认演示入口是桌面端，而不是命令行视觉调试程序。
2. 主界面要持续告诉操作者“下一步该做什么”。
3. 当前不该使用的按钮要自动禁用，避免把演示流程做成调试台。
4. 失败操作要在主界面立刻可见，不能要求用户先展开日志面板。
5. 详细调试信息仍然要保留在 `Activity` 或等价日志区域。

## 非功能边界

- 固定摄像头机位，不在运行中频繁移动。
- 当前只支持单目标人物，不做多目标身份管理。
- Blue Pill 端保持简单，不做视觉识别与目标语义判断。
- 文档和运行说明默认以 Windows 为主。

## 当前范围外

- ReID、上传参考图找人、多目标管理
- 双轴云台、空间定位、距离估计
- 与当前演示主线无关的输入模态或外围硬件

## 验收标准

- 能稳定打开摄像头并看到实时画面。
- 能完成“连接设备 -> 打开相机 -> 选中人物 -> 持续指向”的闭环演示。
- 目标丢失后设备行为符合 `STOP` 或 `CENTER` 配置，不出现随机输出。
- 主界面会展示下一步指引，且错误时能立刻给出用户可理解的反馈。
- Windows 侧串口、摄像头和实机动作验证通过。
