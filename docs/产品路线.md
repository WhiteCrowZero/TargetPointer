# 产品路线

## 当前阶段判断

当前产品已经不是“功能还没闭环的算法样机”，而是“软件主链路已跑通、等待实机收尾的互动演示装置”。

下一阶段不应继续扩边界，而应优先把演示链路做稳、做顺、做清楚。

## 当前优先级

### P1 可重复演示

- 固定摄像头方案收敛
- 独立 5V 供电方案收敛
- Windows 侧完成一次完整闭环复测

### P1 目标选择与状态反馈

- 保持“点击检测框”作为主交互
- 拖框初始化继续保留为兜底方式
- 让观众能更快看懂“当前正在指向谁、系统处于什么状态”

### P2 设备动作质感

- 继续优化舵机分段限速参数
- 降低大偏差时的突兀感和抖动
- 让持续跟随既稳又有可见反馈

### P2 文档与演示材料收束

- 文档围绕桌面端主入口和当前协议保持同步
- 保持 Windows 优先的操作说明
- 清理历史性、重复性说明

## 当前不追求

- 多目标产品化能力
- 身份识别与 ReID
- 复杂人机输入方式
- 双轴云台和空间定位
- 完整消费级硬件产品化
