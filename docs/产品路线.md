# 产品路线

## 产品定位

TargetPointer 当前不是一个通用监控产品，也不是复杂身份识别系统，而是一套**固定摄像头下的互动演示装置**。

它的核心体验是：

- 用户在屏幕中选中一个人
- 系统稳定锁定这个人
- 现实世界里的实体箭头持续指向这个人

这条产品路线强调的是“数字选择驱动物理反馈”，而不是算法堆叠或功能扩张。

## 当前产品故事

当前最适合对外讲述的故事是：

> 在固定摄像头场景下，用户从实时画面中选择一个人物，系统随即驱动实体箭头持续指向该人物，让屏幕中的选择变成现实中的方向反馈。

当前版本重点突出 4 个关键词：

- 直观
- 克制
- 可控
- 可重复

## 当前阶段目标

当前阶段不追求产品化量产，也不追求复杂多场景泛化，而是先把“演示体验”做完整。

已锁定的产品边界：

- 固定摄像头
- 单目标人物
- 单轴水平指向
- 上位机控制
- 实体箭头反馈

## 继续完善的优先级

### P1 演示主流程

- 让演示者可以用最少步骤完成一次完整操作
- 默认入口统一为桌面上位机
- 连接设备、打开摄像头、选人、持续指向这条链路必须稳定

### P1 目标选择体验

- 点击检测框仍然是默认推荐交互
- 手动拖框保留为兜底方式
- 选中后要立即给出清晰锁定反馈

### P1 实体动作气质

- 舵机动作优先追求稳和克制
- 初始大偏差时不能突然猛跳
- 持续跟随时既要安全，也要明显能跟上

### P2 演示界面收束

- 主界面只显示演示必要信息
- 弱化工程调试字段
- 让第一次看到的人也能快速理解系统在做什么

### P2 结构前提说明

- 明确固定摄像头与箭头装置的几何关系要稳定
- 不把手持手机摄像头作为主叙事
- 后续默认推荐固定 USB 摄像头 + 固定箭头底座

## 当前不追求的方向

- 多目标身份管理
- ReID
- 上传参考图找人
- 双轴云台
- 三维空间定位
- 完整产品化硬件定制

## 当前完成度判断

- 软件主链路已经具备可演示形态
- 产品故事已经明确为互动演示装置
- 当前最大短板不在软件功能，而在实机硬件条件不足：
  - 固定摄像头方案未最终落实
  - 独立 5V 供电未落实
  - 实体动作的实机参数还未完成标定

## 下一阶段路线

### 阶段 A：可重复演示

- 固定摄像头方案落地
- 独立 5V 供电落地
- 完成一次完整实机闭环复测

### 阶段 B：演示质感优化

- 继续收窄桌面端 UI
- 优化目标锁定反馈
- 优化箭头动作气质和稳定性

### 阶段 C：结构样机化

- 补齐固定支架、箭头底座和供电整理
- 让整套装置更像一个完整演示样机
