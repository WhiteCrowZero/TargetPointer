# 接口与协议

## 总体说明

“寻音指针”当前只保留一种对外控制接口：上位机通过串口向 Blue Pill 发送文本命令。Blue Pill 解析命令后执行舵机动作，不参与语音识别和目标检测。

## 串口链路参数

- 物理链路：`CH340 USB-TTL -> Blue Pill USART1`
- 波特率：`115200`
- 数据位：`8`
- 校验位：`N`
- 停止位：`1`
- 换行格式：每条命令以 `\n` 结尾，建议上位机统一发送 ASCII 文本

示例：

```text
ANGLE:105
CENTER
STOP
PING
```

## 命令定义

### `ANGLE:<deg>`

- 含义：让舵机转到指定角度。
- 参数：`<deg>` 为十进制整数。
- 推荐范围：`20` 到 `160`。
- 示例：`ANGLE:120`

### `CENTER`

- 含义：让舵机回到中心角。
- 默认中心角：`90°`

### `STOP`

- 含义：停止当前动作并保持当前位置。
- 用途：调试或上位机中断流程时使用。

### `PING`

- 含义：链路探测命令。
- 建议响应：`PONG`

### `TARGET:<name>`

- 含义：可选调试命令，仅用于记录当前目标类别。
- 示例：`TARGET:cup`
- 说明：Blue Pill 可忽略该命令，也可以仅回显用于日志。

### `STATUS?`

- 含义：可选状态查询命令。
- 建议响应：返回当前角度、最近一次命令和解析状态。

## 字段与解析规则

- 命令字母统一大写。
- `ANGLE` 命令只接受整数角度。
- 解析时先按首个 `:` 分隔命令头和参数。
- 多余空格建议由上位机避免；Blue Pill 可在解析前做首尾裁剪。
- 未识别命令应返回错误信息或忽略，并记录到串口日志。

## 异常处理建议

- 角度超出 `20° ~ 160°` 时，Blue Pill 应拒绝执行，或夹紧到安全范围后回传告警。
- 空命令、乱码或不完整命令应直接丢弃。
- 上位机在超时未收到 `PONG` 或状态响应时，应提示串口链路异常。
- 上位机在连续检测不到目标时，不应持续发送随机角度，建议发送 `STOP` 或 `CENTER`。

## 推荐响应格式

Blue Pill 端可返回简单文本，便于调试：

```text
OK:ANGLE:120
OK:CENTER
OK:STOP
PONG
ERR:BAD_CMD
ERR:BAD_ANGLE
```

## 上位机内部数据流

1. 用户说出目标名称。
2. 语音识别模块输出文本结果。
3. 上位机将文本映射为目标类别，例如 `cup`。
4. YOLO 在视频帧中检测该类别，得到目标框。
5. 上位机计算目标框中心点横坐标。
6. 上位机根据中心点偏移换算目标方向，再映射为舵机角度。
7. 上位机发送 `ANGLE:<deg>` 给 Blue Pill。

## 上位机内部建议数据字段

- `target_name`：用户输入的目标名
- `target_class`：送入 YOLO 的类别标识
- `bbox_center_x`：目标框中心横坐标
- `frame_width`：当前画面宽度
- `servo_angle_deg`：最终要发送的角度

这些字段只在上位机内部使用，不需要通过串口逐项传输。
