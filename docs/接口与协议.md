# 接口与协议

## 总体说明

当前系统只有一条对外控制链路：上位机通过串口向 Blue Pill 发送文本命令。人物检测、目标维护和角度计算都发生在上位机；固件只做协议解析、舵机执行和状态回报。

## 串口参数

- 物理链路：`CH340 USB-TTL -> Blue Pill USART1`
- 波特率：`115200`
- 数据格式：`8N1`
- 命令结束：每条命令以 `\n` 结尾
- 建议编码：ASCII

## 当前命令

```text
PING
CENTER
STOP
ANGLE:105
STATE:LOCK
BUZZER:BEEP
LED:ON
LED:OFF
STATUS?
STATUS
```

### `PING`

- 用途：链路探测
- 典型响应：`PONG`

### `ANGLE:<deg>`

- 用途：设置新的目标角
- 参数：十进制整数
- 当前安全范围：`20` 到 `160`
- 典型响应：`OK:ANGLE:<deg>` 或 `ERR:BAD_ANGLE`

### `CENTER`

- 用途：把目标角设回中心角
- 当前中心角：`90`
- 当前行为：渐进回中，不直接猛跳
- 典型响应：`OK:CENTER`

### `STOP`

- 用途：停止继续移动并保持当前位置
- 典型响应：`OK:STOP`

### `LED:ON` / `LED:OFF`

- 用途：废弃兼容命令，只保留解析与响应，不再改变设备状态或 LED 输出
- 典型响应：`OK:LED:DEPRECATED`
- 新代码不要再发送该命令；状态提示统一使用 `STATE:<mode>`

### `STATE:<mode>`

- 用途：同步设备状态，由固件统一驱动红绿 LED 和蜂鸣器
- 支持状态：`IDLE`、`SEARCH`、`LOCK`、`LOST`
- 典型响应：`OK:STATE:LOCK`

### `BUZZER:ON` / `BUZZER:OFF` / `BUZZER:BEEP`

- 用途：手动验证低电平触发蜂鸣器模块
- `BUZZER:ON`：持续拉低蜂鸣器信号，典型响应 `OK:BUZZER:ON`
- `BUZZER:OFF`：释放蜂鸣器信号，典型响应 `OK:BUZZER:OFF`
- `BUZZER:BEEP`：短响 1 声，典型响应 `OK:BUZZER:BEEP`

### `STATUS?` / `STATUS`

- 用途：查询当前设备状态
- 说明：`STATUS` 保留为兼容别名
- 典型响应：

```text
STATUS:ANGLE=90,TARGET=90,ATTACHED=0,LED=OFF,MODE=IDLE,LAST=BOOT,RESULT=OK:IDLE
```

## 当前状态字段

- `ANGLE`：当前输出角
- `TARGET`：当前目标角
- `ATTACHED`：舵机是否已 attach，`1` 为已 attach，`0` 为未 attach
- `LED`：板载 LED 当前状态
- `MODE`：当前设备状态，取值为 `IDLE`、`SEARCH`、`LOCK` 或 `LOST`
- `LAST`：最近一次处理的命令类别
- `RESULT`：最近一次处理结果

## 固件行为约定

- 舵机采用懒 attach，设备启动后默认保持空闲。
- `CENTER` 和 `ANGLE` 都通过配置化步进参数渐进到位。
- `STATE:LOCK` 进入时蜂鸣器短响一声；`STATE:LOST` 进入时短响两声。
- `STATE:IDLE` 和 `STATE:SEARCH` 会立刻静音，并清空未完成的蜂鸣器提示。
- 蜂鸣器信号采用开漏式低电平触发：响时 `PB12` 下拉到 GND，不响时释放为高阻输入，适配 5V 低电平触发模块。
- 红绿 LED 完全由 `STATE` 状态机驱动，`LED:ON/OFF` 不再覆盖状态机输出。
- 未识别命令返回 `ERR:BAD_CMD`。
- 角度越界返回 `ERR:BAD_ANGLE`。

## 上位机建议

- 启动或重连后先发送 `STATUS?`，同步真实设备状态。
- 使用 `STATE` 同步完整设备状态：选择目标前发送 `STATE:SEARCH`，锁定或短时重获中发送 `STATE:LOCK`，连续丢失后发送 `STATE:LOST`，退出或断开前发送 `STATE:IDLE`。
- 目标连续丢失时应转入 `STOP` 或 `CENTER`，不要继续发送随机角度。
- UI 层只显示面向操作者的短错误信息，详细串口内容保留在日志里。
