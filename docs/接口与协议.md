# 接口与协议

## 总体说明

当前系统只有一条对外控制链路：上位机通过串口向 Blue Pill 发送文本命令。人物检测、目标维护和角度计算都发生在上位机；固件只做协议解析、舵机执行和状态回报。

## 串口参数

- 物理链路：`CH340 USB-TTL -> Blue Pill USART1`
- 波特率：`115200`
- 数据格式：`8N1`
- 命令结束：每条命令以 `\n` 结尾
- 建议编码：ASCII

## 当前命令

```text
PING
CENTER
STOP
ANGLE:105
LED:ON
LED:OFF
STATUS?
STATUS
```

### `PING`

- 用途：链路探测
- 典型响应：`PONG`

### `ANGLE:<deg>`

- 用途：设置新的目标角
- 参数：十进制整数
- 当前安全范围：`20` 到 `160`
- 典型响应：`OK:ANGLE:<deg>` 或 `ERR:BAD_ANGLE`

### `CENTER`

- 用途：把目标角设回中心角
- 当前中心角：`90`
- 当前行为：渐进回中，不直接猛跳
- 典型响应：`OK:CENTER`

### `STOP`

- 用途：停止继续移动并保持当前位置
- 典型响应：`OK:STOP`

### `LED:ON` / `LED:OFF`

- 用途：控制板载 LED 阶段提示
- 典型响应：`OK:LED:ON` / `OK:LED:OFF`

### `STATUS?` / `STATUS`

- 用途：查询当前设备状态
- 说明：`STATUS` 保留为兼容别名
- 典型响应：

```text
STATUS:ANGLE=90,TARGET=90,ATTACHED=0,LED=OFF,LAST=BOOT,RESULT=OK:IDLE
```

## 当前状态字段

- `ANGLE`：当前输出角
- `TARGET`：当前目标角
- `ATTACHED`：舵机是否已 attach，`1` 为已 attach，`0` 为未 attach
- `LED`：板载 LED 当前状态
- `LAST`：最近一次处理的命令类别
- `RESULT`：最近一次处理结果

## 固件行为约定

- 舵机采用懒 attach，设备启动后默认保持空闲。
- `CENTER` 和 `ANGLE` 都通过配置化步进参数渐进到位。
- 未识别命令返回 `ERR:BAD_CMD`。
- 角度越界返回 `ERR:BAD_ANGLE`。

## 上位机建议

- 启动或重连后先发送 `STATUS?`，同步真实设备状态。
- 跟踪锁定或重获中可发送 `LED:ON`，空闲或丢失时发送 `LED:OFF`。
- 目标连续丢失时应转入 `STOP` 或 `CENTER`，不要继续发送随机角度。
- UI 层只显示面向操作者的短错误信息，详细串口内容保留在日志里。
