# 接口与协议

## 总体说明

当前系统只保留一种对外控制接口：上位机通过串口向 Blue Pill 发送文本命令。协议的核心用途是方向控制与设备状态控制，而不是目标识别。

明确边界：

- 上位机负责人物检测、目标初始化、持续定位和角度计算。
- Blue Pill 负责命令解析、舵机动作执行和状态回报。
- 当前系统不是类别物体指向系统，固件也不处理任何 `person` 以外的目标语义。

## 串口链路参数

- 物理链路：`CH340 USB-TTL -> Blue Pill USART1`
- 波特率：`115200`
- 数据位：`8`
- 校验位：`N`
- 停止位：`1`
- 换行格式：每条命令以 `\n` 结尾
- 建议编码：ASCII

示例：

```text
ANGLE:105
CENTER
STOP
PING
STATUS?
```

## 命令定义

### `ANGLE:<deg>`

- 含义：让舵机转到指定角度。
- 参数：`<deg>` 为十进制整数。
- 推荐范围：`20` 到 `160`。
- 用途：上位机根据人物目标的水平偏移计算后发送。

### `CENTER`

- 含义：让舵机回到中心角。
- 默认中心角：`90°`
- 用途：初始化、人工复位或丢失策略。

### `STOP`

- 含义：停止当前动作并保持当前位置。
- 用途：目标连续丢失、人工中断或调试。

### `PING`

- 含义：链路探测命令。
- 建议响应：`PONG`

### `STATUS?`

- 含义：查询设备状态。
- 当前固件响应格式：`STATUS:ANGLE=90,LAST=CENTER,RESULT=OK:CENTER`

## 字段与解析规则

- 命令字母统一大写。
- `ANGLE` 只接受整数角度。
- 解析时按首个 `:` 分隔命令头和参数。
- 固件对未识别命令返回错误信息。
- 固件不接收目标类别、目标姓名或视觉层语义字段。

## 异常处理建议

- 角度超出安全范围时返回 `ERR:BAD_ANGLE`。
- 空命令、乱码或不完整命令直接丢弃或返回 `ERR:BAD_CMD`。
- 上位机在未收到 `PONG` 或状态响应时提示串口链路异常。
- 上位机在目标连续丢失时不发送随机角度，应转入 `STOP` 或 `CENTER`。

## 推荐响应格式

```text
OK:ANGLE:120
OK:CENTER
OK:STOP
PONG
STATUS:ANGLE=90,LAST=CENTER,RESULT=OK:CENTER
ERR:BAD_CMD
ERR:BAD_ANGLE
```

## 上位机内部数据流

1. 摄像头提供实时视频帧。
2. YOLO 检测画面中的人物，输出候选框。
3. 用户选择一个人物候选框，完成初始化。
4. 上位机持续更新该目标位置。
5. 上位机根据目标框中心点横坐标计算舵机角度。
6. 上位机发送 `ANGLE:<deg>`、`STOP` 或 `CENTER` 给 Blue Pill。

## 上位机内部建议字段

- `selected_person_bbox`
- `bbox_center_x`
- `frame_width`
- `servo_angle_deg`
- `missed_frames`
- `tracking_state`

这些字段只在上位机内部使用，不需要通过串口逐项传输。
