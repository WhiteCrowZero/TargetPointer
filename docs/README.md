# 文档导航

`docs/` 目录只服务于当前单主线版本：固定摄像头视角下，由上位机完成 YOLO 人物检测、目标初始化、持续定位与串口控制，Blue Pill 负责执行 SG90 单轴水平指向。

## 推荐阅读顺序

1. [项目概述.md](项目概述.md)：项目定位、场景和唯一主线。
2. [产品路线.md](产品路线.md)：产品故事、当前阶段目标和后续完善优先级。
3. [需求说明.md](需求说明.md)：当前版本的范围、边界和验收重点。
4. [架构设计.md](架构设计.md)：上位机控制链路、YOLO 初始化和固件职责分层。
5. [接口与协议.md](接口与协议.md)：方向控制与状态查询协议。
6. [硬件规格.md](硬件规格.md)：固定摄像头、Blue Pill、SG90 与供电约束。
7. [硬件调试计划.md](硬件调试计划.md)：实机调试顺序。
8. [硬件调试记录.md](硬件调试记录.md)：当前硬件盘点与实测记录模板。
9. [开发TODO.md](开发TODO.md)：按演示优先级推进的分日任务。

## 维护约定

- 文档统一描述为“固定摄像头下的实体人物指向系统”。
- 当前系统只关注单目标人物，不保留旧的类别物体指向叙事。
- 目标识别逻辑全部在上位机，固件不做识别判断。
- 当前文档默认以“代码已实现的软件链路 + 尚待完成的实机验证”来描述状态。
- 变更硬件边界、串口协议或视觉主线时，先更新这里的文档再改代码。
