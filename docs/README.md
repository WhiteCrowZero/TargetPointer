# 文档导航

`docs/` 只描述当前有效的单主线版本：固定摄像头、单目标人物、上位机感知与控制、Blue Pill 执行 SG90 单轴水平指向。

## 推荐阅读顺序

1. [项目概述.md](项目概述.md)
   产品定位、当前范围、推荐演示流程。
2. [需求说明.md](需求说明.md)
   当前版本必须满足的功能、交互和验收标准。
3. [架构设计.md](架构设计.md)
   桌面端、视觉运行时、串口层和固件的职责边界。
4. [接口与协议.md](接口与协议.md)
   当前串口协议、状态字段和推荐异常处理。
5. [硬件规格.md](硬件规格.md)
   当前硬件方案、供电边界和结构约束。
6. [硬件调试计划.md](硬件调试计划.md)
   Windows 侧实机联调顺序。
7. [硬件调试记录.md](硬件调试记录.md)
   记录模板和待验证项。
8. [产品路线.md](产品路线.md)
   下一阶段优先级，不重复描述当前实现细节。
9. [开发TODO.md](开发TODO.md)
   当前剩余事项和短期行动项。

## 维护约定

- `docs/项目概述.md` 是当前范围和产品叙事的总入口。
- `docs/需求说明.md` 只写要求和验收，不重复展开架构细节。
- `docs/架构设计.md` 只写模块职责和数据流，不重复产品故事。
- `docs/接口与协议.md` 只写当前代码实际支持的协议和状态字段。
- Windows 是权威运行环境；涉及摄像头、串口、烧录、GUI 的验证结论默认以 Windows 为准。
