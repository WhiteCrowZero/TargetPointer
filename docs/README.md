# 文档导航

这个项目很小，文档只保留当前有效主线，避免同一件事分散在太多文件里。

## 当前保留文档

1. [项目概述.md](项目概述.md)
   产品目标、范围边界、软件架构、实时语音、报告和短期优先级。
2. [接口与协议.md](接口与协议.md)
   上位机和 Blue Pill 固件之间的串口命令、状态字段和行为约定。
3. [硬件接线与验证.md](硬件接线与验证.md)
   BOM、引脚、供电、接线顺序、Windows 实机验证流程和常见问题。
4. [硬件调试记录.md](硬件调试记录.md)
   只记录真实环境或镜像侧变更的验证结论，不再拆成单独计划文档。

## 维护约定

- `docs/项目概述.md` 是范围和架构唯一入口。
- 硬件、供电、引脚、接线和验证流程统一维护在 `docs/硬件接线与验证.md`。
- `hardware/README.md` 只放硬件资产索引和最终表格，避免与 docs 重复展开。
- Windows 是串口、摄像头、烧录、GUI 和 LiveKit 实测的权威环境。
- WSL 镜像里的检查只能说明代码和文档格式层面没有明显问题，不能替代实机验证。
