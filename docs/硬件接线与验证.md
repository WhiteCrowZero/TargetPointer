# 硬件接线与验证

本文是当前硬件方案的唯一实操入口，默认在 Windows 工作区执行烧录、串口、摄像头和实机验证。当前方案不再使用屏幕，状态提示只依赖红绿 LED 和蜂鸣器。

## 最终硬件组合

- `Blue Pill (STM32F103C8)`
- `SG90` 舵机
- 绿 LED、红 LED 和限流电阻
- 低电平触发蜂鸣器模块
- `CH340 USB-TTL`
- `ST-LINK V2`
- 独立 5V 舵机供电或短时原型供电辅助
- 面包板、杜邦线、箭头固定件或舵机支架

建议补齐：固定摄像头支架、稳定 5V 舵机电源、刚性更好的舵机支架。旧定位接收器、旧标签附件和与当前主线无关的外设不纳入本版本。

## 引脚总表

| Blue Pill 引脚 | 模块 | 说明 |
| --- | --- | --- |
| `PA8` | `SG90 signal` | 舵机 PWM |
| `PB12` | `Buzzer SIG` | 蜂鸣器信号，低电平触发 |
| `PB0` | 绿 LED 正极 | 串联电阻后接 LED 正极，负极接 GND |
| `PB1` | 红 LED 正极 | 串联电阻后接 LED 正极，负极接 GND |
| `PA9` | `CH340 RX` | Blue Pill 串口 TX |
| `PA10` | `CH340 TX` | Blue Pill 串口 RX |
| `PA13/PA14` | `ST-LINK SWDIO/SWCLK` | 下载调试 |
| `PC13` | 板载 LED | 与绿 LED 同步 |

`PA13/PA14` 保留给 SWD，不复用；`PA8` 优先保留给舵机 PWM；`PB0/PB1/PB12` 保留给外接状态提示硬件。

## 接线顺序

1. 先接 `ST-LINK`：`3.3V/GND/SWDIO/SWCLK`。
2. 接 `CH340`：`TX -> PA10`、`RX -> PA9`、`GND -> GND`。
3. 接 `SG90`：信号线接 `PA8`，电源接独立 `5V`，地线与 Blue Pill 共地。
4. 接蜂鸣器：`SIG -> PB12`、`VCC -> 5V`、`GND -> GND`。
5. 接 LED：`PB0/PB1 -> 电阻 -> LED 正极`，LED 负极接 `GND`。

所有电源地必须共地：Blue Pill、CH340、SG90、电源模块、蜂鸣器和 LED 地线都要连到同一个 GND 网络。

## 供电边界

| 供电轨 | 用途 | 规则 |
| --- | --- | --- |
| `3.3V` | Blue Pill 逻辑 | 不给 SG90 供电 |
| `5V` | SG90、蜂鸣器模块 | 舵机优先独立供电 |
| `USB` | CH340 串口 | 只作为通信和短时原型辅助 |

SG90 启动和堵转瞬间电流较高，最终展示不要依赖 Blue Pill `3.3V` 或 CH340 弱供电。若舵机动作时串口断线、Blue Pill 复位或蜂鸣器异常，优先检查 5V 电源余量和共地。

## 上电前检查

- `PA13/PA14` 没有被 LED 或蜂鸣器占用。
- `PA9/PA10` 没有接反；若串口无响应，优先互换 CH340 的 TX/RX。
- SG90 没有从 Blue Pill `3.3V` 直接取电。
- LED 串了限流电阻，不能直接把 LED 接在 IO 和 GND 之间。
- 蜂鸣器模块按低电平触发处理；固件响的时候把 `PB12` 下拉到 GND，不响时释放为高阻输入。

## 烧录与最小串口验证

当前默认串口是 `COM4`。如果 Windows 设备管理器显示不同串口号，再把命令里的 `COM4` 换成实际端口。

```bash
uv sync
uv run pio run --project-dir firmware
uv run pio run --project-dir firmware -t upload
uv run python -m serial.tools.list_ports -v
uv run python scripts/pointer_serial_cli.py --port COM4 ping
uv run python scripts/pointer_serial_cli.py --port COM4 status
```

预期：

- `ping` 返回 `PONG`
- `status` 返回 `STATUS:...MODE=IDLE...`
- 上电后红绿 LED 都熄灭，蜂鸣器不响

## 蜂鸣器单独验证

先单独确认蜂鸣器模块能被固件控制，再测状态机：

```bash
uv run python scripts/pointer_serial_cli.py --port COM4 buzzer off
uv run python scripts/pointer_serial_cli.py --port COM4 buzzer beep
uv run python scripts/pointer_serial_cli.py --port COM4 buzzer on
uv run python scripts/pointer_serial_cli.py --port COM4 buzzer off
```

预期：

- `buzzer off`：不响。
- `buzzer beep`：短响 1 声。
- `buzzer on`：持续响。
- 最后一次 `buzzer off`：立刻静音。

## 状态硬件验证

按顺序手动发送：

```bash
uv run python scripts/pointer_serial_cli.py --port COM4 state search
uv run python scripts/pointer_serial_cli.py --port COM4 state lock
uv run python scripts/pointer_serial_cli.py --port COM4 state lost
uv run python scripts/pointer_serial_cli.py --port COM4 state idle
```

预期现象：

| 命令 | 绿 LED | 红 LED | 蜂鸣器 |
| --- | --- | --- | --- |
| `state search` | 慢闪 | 灭 | 不响，清空未完成提示 |
| `state lock` | 常亮 | 灭 | 短响 1 声 |
| `state lost` | 灭 | 常亮 | 短响 2 声 |
| `state idle` | 灭 | 灭 | 不响，清空未完成提示 |

旧 `LED:ON/OFF` 命令已经废弃，只用于兼容旧脚本解析。它会返回 `OK:LED:DEPRECATED`，不会改变红绿 LED 或设备状态。

## 舵机验证

```bash
uv run python scripts/pointer_serial_cli.py --port COM4 center
uv run python scripts/pointer_serial_cli.py --port COM4 angle 60
uv run python scripts/pointer_serial_cli.py --port COM4 angle 120
uv run python scripts/pointer_serial_cli.py --port COM4 stop
```

检查点：

- 舵机在 `20° ~ 160°` 安全范围内动作。
- `status` 中 `ANGLE` 和 `TARGET` 字段随固件当前输出角变化。
- 舵机动作时 Blue Pill 不复位，串口不断线。
- 当前摄像头画面是镜像画面，上位机已把视觉角度反向：画面左侧目标应让输出角变大，画面右侧目标应让输出角变小。若实机方向仍反，先不要改固件，优先复查摄像头镜像和上位机版本。

## 整机验证

```bash
uv run python scripts/pointer_desktop_app.py --port COM4 --camera 0 --camera-backend msmf --model yolov8n.pt
```

检查顺序：

1. 连接串口后设备应进入 `SEARCH`：绿 LED 慢闪。
2. 打开摄像头，点击检测到的人物框。
3. 锁定后设备进入 `LOCK`：绿 LED 常亮，蜂鸣器响 1 声。
4. 目标丢失后设备进入 `LOST`：红 LED 常亮，蜂鸣器响 2 声。
5. 断开串口或退出时设备进入 `IDLE`：红绿 LED 熄灭。

## 常见问题

| 现象 | 优先排查 |
| --- | --- |
| 串口无响应 | COM 口是否正确、TX/RX 是否接反、是否共地 |
| 蜂鸣器一直响 | 先执行 `buzzer off`，再查模块是否低电平触发、`SIG` 是否接到 `PB12`、GND 是否共地 |
| LED 不亮 | LED 极性、限流电阻、`PB0/PB1` 是否接错 |
| `LED:ON/OFF` 不改变灯 | 这是预期行为，改用 `state search/lock/lost/idle` |
| 舵机抖动或复位 | SG90 是否独立 5V 供电、箭头是否过重、GND 是否共地 |
| 舵机方向相反 | 确认摄像头是否镜像、上位机是否包含 2026-04-23 后的角度反向修复 |

## 结构约束

- 舵机和箭头应固定在同一刚性底座上，避免箭头晃动造成视觉误判。
- 箭头尽量轻，重心靠近舵机轴，避免 SG90 长时间堵转。
- 摄像头保持固定高度和固定视角；调参后不要随意移动机位。
- CH340、ST-LINK、舵机线和 LED 线要留出应力余量，避免演示时拉扯导致接触不良。
